GT LEARN #372 🪜 Super Pow MEDIUM Math · Recursion exp: 1024
📝 Problem Statement
Calculate ab mod 1337 where a is a positive integer and b is an extremely large positive integer given as an array of digits (most-significant first). Example: if b = [1,0,0], that means b = 100, so we compute a100 mod 1337. Direct exponentiation is impossible — b can have thousands of digits!
🧠 Solving Approach
Use the identity: a[d1,d2,...,dn] = adn × (a[d1,d2,...,dn-1])10. Recursively peel off the last digit. Apply modular exponentiation at every step (Python's built-in pow(base, exp, mod)) to keep numbers small.
🎯 Core Concepts & Base Case
Base Case: Empty array b = [] → return 1 (since a0 = 1).
Recurrence: superPow(a, b) = (alast × superPow(a, b[:-1])10) mod 1337
Why mod at every step: (x × y) mod m = ((x mod m) × (y mod m)) mod m — keeps intermediate values bounded.
⏱ Time: O(n) 💾 Space: O(n)
