GT LEARN #13 🏛️ Roman to Integer EASY String Hash Map Math
s= exp: 1994

📝 Problem Statement

Given a Roman numeral string s, convert it to its integer value. Roman has 7 symbols: I=1, V=5, X=10, L=50, C=100, D=500, M=1000. Usually we just add up the values left to right. But for subtractive pairs (like IV=4, IX=9, XL=40), the smaller symbol comes before a larger one and must be subtracted instead of added.

🧠 Solving Approach

One-pass with lookahead. Walk through every character. At each position, compare its value to the next character's value. If s[i] < s[i+1] → it's the small half of a subtractive pair → subtract. Otherwise → add. Last character is always added (no next neighbour to compare).
🎯 Key trick: No need to detect 2-char pairs explicitly. Just one rule: "smaller before larger = subtract; otherwise add". Both halves of IV get processed correctly — I is subtracted (−1), V is added (+5), net = +4. ✓
♻️ Loop pattern: Single for i in range(len(s)). At the last index, no neighbour exists → always add.
🏁 Termination: When i == len(s), loop exits → return total.
⏱ Time: O(n) 💾 Space: O(1) — fixed dict
