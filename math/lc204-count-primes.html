GT LEARN #204 🔢 Count Primes (Sieve of Eratosthenes) MEDIUM Math Sieve Array
n= exp: 8

📝 Problem Statement

Given an integer n, return the number of prime numbers strictly less than n (i.e., count primes in [2, n-1]). A prime has exactly 2 divisors: 1 and itself. Constraint: 0 ≤ n ≤ 5·10⁶.

🧠 Solving Approach — Sieve of Eratosthenes

Build a boolean array is_prime[0..n-1] defaulted to True. Mark 0,1 as False. Then for every i from 2 up to √n: if is_prime[i] is still True, mark all its multiples starting from i·i as composite. Finally, count remaining True cells from index 2 onward.
🎯 Why start marking at i·i? All smaller multiples (2i, 3i, ..., (i−1)i) are already marked by smaller primes. E.g., when i=5, we skip 10 (done by i=2), 15 (done by i=3), 20 (done by i=2) — start fresh at 25.
♻️ Why outer loop only goes up to √n? If a number x < n is composite, it has at least one factor ≤ √x ≤ √n. So all composites get caught by the time we finish i = √n. Beyond that, nothing new to mark.
🏁 Termination: Two-phase. Phase 1 = sieve marking (outer i loop). Phase 2 = count loop over [2, n).
⏱ Time: O(n·log log n) 💾 Space: O(n) — bool array
