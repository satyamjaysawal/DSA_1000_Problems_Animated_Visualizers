GT LEARN #1334 🗺️ Find the City With Smallest Number of Neighbors MEDIUM Floyd-Warshall Graph
exp: 3
📝 Problem Statement
Given n cities (numbered 0..n-1) and weighted bidirectional edges[i] = [from, to, weight] plus a distanceThreshold, find the city with the smallest number of cities reachable within the threshold. Tie-breaker: return the city with the greatest number.
🧠 Solving Approach
Use Floyd-Warshall all-pairs shortest path. Initialize distance matrix with edge weights and INF elsewhere. For each intermediate k, relax every (i,j) via dist[i][k] + dist[k][j]. Then count reachable neighbors per city under the threshold and pick the best.
🎯 Core Idea
For every triple (k, i, j): if going i → k → j is shorter than current i → j, update it. After all k, every dist[i][j] is the true shortest path.
🔁 Loop Order Matters
Outer loop is k (intermediate node). Inner loops are i and j (source/destination). Wrong order = wrong answer!
🏆 Tie-breaker
If two cities have the same minimum count, return the one with larger index. Loop forward and use <= on count comparison.
⏱ Time: O(n³) 💾 Space: O(n²)
