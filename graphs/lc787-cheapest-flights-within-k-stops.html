GT LEARN #787 ✈️ Cheapest Flights Within K Stops MEDIUM Modified Dijkstra Min-Heap Graph
exp: 200
📝 Problem Statement
n sheher hain, weighted directed flights diye gaye hain [from, to, price] format mein. src se dst tak at most k stops (intermediate cities) ke saath jaane ka cheapest total price nikalo. Possible nahi ho toh -1.
🧠 Solving Approach
Modified Dijkstra with state = (cost, node, stops). Min-heap se sabse sasta path pop karte hain. Stops > k toh prune. Same (node, stops) pe agar already cheaper aaya hai toh bhi prune.
🎯 Key Insight
Plain Dijkstra fail karta hai kyunki cheaper-but-more-stops vs costlier-but-fewer-stops trade-off hai. Stops ko state mein include karna padta hai.
🛬 Stop Counting
k stops = k intermediate cities. Toh edge count k+1 tak allowed. Stops jab k+1 ho jaaye toh prune.
🏆 Early Exit
Min-heap ki property: jab dst pop hota hai, woh apni cheapest valid price ke saath nikalta hai (kyunki priority = cost). Turant return!
⏱ Time: O(E·k·log(E·k)) 💾 Space: O(n·k)
🧪 Test Cases

→ 200
