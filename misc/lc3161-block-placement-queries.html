📝 Problem Statement
Infinite number line. Two query types: [1, x] = place an obstacle at position x (positions are unique). [2, x, sz] = ask: can we place a block of size sz fully inside [0, x] such that no obstacle lies strictly inside the block? The block's right edge must touch x or end before — and the block must fit in some obstacle-free gap ending at-or-before x.
🧠 Solving Approach
Maintain a sorted set of obstacle positions (always includes 0). Track the maximum gap ending at each obstacle — the gap from previous obstacle to this obstacle. For a query (x, sz): find the largest obstacle ≤ x, check if max-gap up to that point ≥ sz, OR if the partial gap (from previous obstacle to x) ≥ sz.
🎯 Core Concepts
Obstacles set: sorted, starts as {0}. Insertion = O(log n) (uses bisect).
maxGap[i]: largest gap ending ≤ obstacle i (segment-tree max).
Query answer: True iff maxGap[predecessor of x] ≥ sz OR (x − predecessor) ≥ sz.
⏱ Time: O(q · log q) 💾 Space: O(q)
